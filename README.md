# 🖨 CoolPrint v7.5

**Бот-Оркестратор для умного управления печатью через Telegram.**
Отправил PDF → Настроил параметры → Получил live-статус и автоматическое охлаждение.

---

## ✨ Основные возможности

*   🚀 **Умная нарезка:** Автоматическое разделение больших файлов на пачки с паузами для охлаждения.
*   🔄 **Smart Recovery:** Если печать прервалась (замятие, кончилась бумага, сбой сети), бот использует SNMP для точного определения последней напечатанной страницы и предлагает продолжить с нужного места.
*   🌡 **Контроль перегрева:** Настраиваемая пауза между пачками для предотвращения износа принтера.
*   📑 **Поддержка Booklet:** Режим печати брошюр (тетрадками).
*   📊 **Live-мониторинг:** Реальное отображение прогресса печати (страница X из Y) прямо в Telegram.
*   📦 **Многозадачность:** Поддержка нескольких копий и кастомных диапазонов страниц.

## 🛠 Установка

1.  **Клонируйте репозиторий:**
    ```bash
    git clone https://github.com/abboskhonf/coolprint.git
    cd coolprint
    ```
2.  **Создайте виртуальное окружение и установите зависимости:**
    ```bash
    python -m venv .venv
    source .venv/bin/activate  # Windows: .venv\Scripts\activate
    pip install pypdf pysnmp-lextudio aiohttp aiogram
    ```
3.  **Windows:** Положите `SumatraPDF.exe` (portable версию) в корень проекта.

## ⚙️ Настройка (config.ini)

Отредактируйте `config.ini` перед запуском:

```ini
[telegram]
token = 123456789:ABC...       ; Токен от @BotFather
chat_id = 987654321             ; Ваш ID (или ID группы)
download_timeout = 300          ; Таймаут загрузки файлов (сек)

[printer]
ip = 192.168.1.100              ; IP принтера в локальной сети
name = Canon iR2425             ; Имя принтера в системе (CUPS или Windows)

[snmp]
community = public              ; SNMP Community (обычно public)
port = 161
wake_timeout_sec = 360          ; Время ожидания прогрева принтера

[print]
batch_size = 64                 ; Размер пачки (страниц)
cooldown_minutes = 25           ; Пауза для охлаждения (мин)
chunks_per_batch = 4            ; На сколько частей делить одну пачку для стабильной передачи
```

## 📖 Использование

### Команды
| Команда | Описание |
| :--- | :--- |
| `/start` | Запуск бота и краткая справка |
| `/status` | Текущий статус принтера (готово/печать/ошибка) + счётчик страниц |
| `/stop` | Остановить печать после завершения текущей пачки |
| `/cancel` | Отменить текущую настройку или сбросить состояние |

### Процесс печати
1.  Отправьте PDF-файл боту.
2.  С помощью инлайн-кнопок настройте:
    *   **Диапазон страниц** (например, `1-50`).
    *   **Дуплекс** (односторонняя, по длинному/короткому краю).
    *   **Количество копий**.
    *   **Размер пачки** и время охлаждения.
3.  Нажмите **🚀 Запустить**.
4.  Следите за прогрессом. Если принтер сообщит о проблеме (нет бумаги), бот пришлет уведомление и будет ждать устранения.

## 📂 Структура проекта

*   `main.py` — Точка входа, запуск бота.
*   `handlers.py` — Логика обработки сообщений и кнопок Telegram.
*   `print_job.py` — Главный цикл печати, управление очередью и состояниями.
*   `print_service.py` — Взаимодействие с системными принтерами (SumatraPDF/CUPS).
*   `pdf_processor.py` — Нарезка, обработка и подготовка PDF-файлов.
*   `snmp_manager.py` — Низкоуровневый мониторинг принтера через SNMP.
*   `config.py` — Загрузка и валидация настроек.
*   `core.py` — Общие модели данных, клавиатуры и утилиты.
*   `state_manager.py` — Сохранение состояния заданий для восстановления после сбоев.

---

## 🛡 Безопасность и Лимиты

*   Бот работает только с `chat_id`, указанным в конфиге.
*   Максимальный размер файла через Telegram — **20 МБ**.
*   Для больших файлов используйте локальный запуск: `python main.py path/to/file.pdf`.
